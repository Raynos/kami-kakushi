# kami-domain-reference

What the game IS, in the departing principal's head: the math models,
the setting canon, the visual contract, and the jargon. This skill is a
MAP — living canon stays in the living docs it points at; generated
numbers stay in the generated docs that own them.

## The number rule (read this first)

Never hardcode a balance value you read here or anywhere else. Many
balance constants are `export let` cockpit levers (ADR-134 — the human
tunes live, agents transcribe) and values drift by design. The teaching
form is always: **read the current value of `CONSTANT_NAME` in
`src/core/content/balance.ts`** (or `src/core/constants.ts`). Measured
outcomes (medians, win-rates) come from generated docs — cite them as
generated and regen to refresh. Same for `SCHEMA_VERSION` /
`APP_GENERATION`: read `src/core/constants.ts`, never quote a number
from a doc (a co-agent bump landed mid-2026-07-18; docs lag).

## 1. The pacing model — tiers, rungs, phases

- **7 tiers `T0..T6`** (`TierId`, `src/core/content/tiers.ts`):
  Estate-household → Estate-land → Valley → Region → Castle Town →
  Domain → Edo. `TIER_NAMES` there is the single source for tier
  naming (AC-21). Tiers REPLACE prestige resets — NO-PRESTIGE is hard
  canon (story-bible constraints, `docs/story-bible/01-laws.md` §0.9).
- **8 rungs `R0..R7` per tier.** After R7 + the capstone flag, Phase 2
  opens (`phaseOf`, `src/core/ranks.ts`).
- **Rung progress = an authored HIDDEN requirement list, NOT points.**
  The old `rungThreshold` points meter is DELETED (FB-121/ADR-137;
  AGENTS.md was synced 2026-07-18) — only two stale comments still
  name it (`balance-sim.ts:61`, `telemetry/milestones.test.ts:12`);
  never re-derive from those. Requirements are authored
  in `src/core/content/narrative/requirements.md`, compiled by
  `pnpm run gen:narrative` into `requirements.gen.ts`, re-exported by
  `src/core/content/requirements.ts`. The player sees only a rounded
  integer percent; the SAME pure fn (`rungProgress`,
  `src/core/ranks.ts`) feeds the bar, the sim, and the gate (AC-6).
- **Promotion banks; the player triggers it.** A ready rung waits until
  the player fires the VN rung beat: `begin_rung_beat` →
  `choose_rung_option` → `applyPromotion` (`src/core/ranks.ts` — the
  SOLE place a rung advances). Promotion refills satiety, never HP.
- **Signed pacing bands** (constant NAMES in
  `src/core/content/balance.ts` — read current values there):
  - `T0_PACING_BAND_MIN` / `T0_PACING_BAND_MAX` — wall-minutes per T0
    climb rung (max re-signed by the human 2026-07-13 for the sickroom
    mend lane).
  - `RUNG_WALL_FLOOR_MIN` — the T1+ per-rung floor; T0 is exempt
    ("quick but not easy").
  - `PHASE2_PHASE1_RATIO_MIN/MAX` — Phase 2 wall-time ≈ Phase 1
    (ADR-133/HD-19), gated only for built Phase-2 tiers.
- **Wall model (ADR-148):** a TIMED intent costs duration + cooldown
  from the core timing table; an instant intent costs one
  `AUTO_REPEAT_MS` heartbeat. Measured medians (e.g. R0 ≈ 4.3 min →
  R7 ≈ 96.1 min, cum ≈ 184.5 min for the greedy T0 arc) are per
  `docs/content/t0-pacing.md` — GENERATED by `pnpm run balance:report`
  (`src/scripts/balance-sim.ts`); regen to refresh, never quote stale.
- **Phase 2** = the post-capstone deed grind to Estate grade EXCELLENT
  (`ESTATE_BANDS.excellent`, balance.ts) → ascension.

Sim methodology, fingerprints, band-law, and the ADR-132 commit
protocol are owned by **kami-balance-analysis-toolkit** — do not touch
balance magnitudes without it.

## 2. Combat math (`src/core/combat.ts`)

- **5 attributes** STR/AGI/INT/SPD/LUCK, all start at `ATTR_BASE`;
  +1 point per `ATTR_POINTS_PER_LEVELS` character levels. Character
  level is combat-XP-fed only: kill grants `mob.level × COMBAT_XP_K`;
  integer 13/10-growth curve from `COMBAT_XP_BASE` (balance.ts).
- **MC derived stats** (`mcCombatStats`, combat.ts):
  - `attackPower = max(1, round((weapon.baseAttack + 1.2·STR) ·
    stanceAtk · satietyRate · durabilityMult · intMult))`, where
    `intMult` applies only vs bestiary-KNOWN foes
    (`INT_KNOWN_DMG_COEFF`).
  - Fights start from CARRIED HP — no free refill (ADR-050), no
    auto-heal; mend only via the sickroom lane (below).
  - Accuracy/evasion/crit/speed all derive linearly from AGI/SPD/LUCK
    — read the coefficients in combat.ts, they are the design levers
    tests assert (AGENTS.md test discipline: assert the lever, not the
    collapsed win-rate).
- **Per-swing exchange:** `hitChance = clamp(acc/(acc+eva), .15, .95)`;
  ±`DMG_VARIANCE` variance; crit ×1.5; damage floor
  `max(1, DAMAGE_FLOOR_FRAC·attackPower)` applied once, last — chip
  damage always lands.
- **Mob curve:** ONE linear-in-level rule per stat (combat.ts);
  archetype knobs live in `enemies.ts`.
- **TWO win-rate lenses** (combat.ts header): `analyticWinRate` =
  closed-form race-to-kill, kept for the M6 gate; the DISPLAYED
  forecast is `sampledWinRate` — `FORECAST_SAMPLES` (400) samples over
  the FIXED seed `FORECAST_SEED` (+ `mob.level·7919`), so every player
  sees the same number for the same stat-state. AC-6: the forecast is
  the same pure fn the fight uses — a hurt fighter's shown win-rate
  drops for free. Never "fix" a forecast by forking the fn.
- **Signed combat bands** (balance.ts, `CURVE_*` constants):
  first-fight (monkey@L1) win-rate in the humbling-but-winnable
  20–35% band (`CURVE_FIRST_FOE_WR_MIN/MAX`); choice band; dead/
  trivial caps; `CURVE_MASTERY_MIN_KILLS`. Marked provisional — tune
  by playtest, verdict by sim.
- **Stances** (`STANCE_MODS`): jodan hits harder/takes more · chudan
  identity · gedan the reverse — glass-cannon↔tank; no stance strictly
  dominated BECAUSE HP carries between fights.
- **Loss shape (LOCKED):** soft-setback only — a loss drops
  `LOSS_COIN_FRAC` of CARRIED coin + `LOSS_MATERIAL_FRAC` of carried
  rice; BANKED kura contents safe; koku never carried, never lost;
  never levels/gear. Defeat costs `SICKROOM_DAYS_LOST` days.
  Auto-retreat at `AUTO_RETREAT_FRAC` HP (per-turn check — a killing
  blow past the threshold still loses).
- **Durability:** `DURABILITY_BANDS` Pristine/Worn/Battered/Broken
  attackPower multipliers (never auto-unequip); flat per-fight wear
  (`DURABILITY_WEAR_PER_FIGHT`); repair via `REPAIR_WOOD_COST` /
  `REPAIR_COIN_COST` — the fight→coin→repair→fight upkeep loop.
- **Sickroom lane (ADR-164/ADR-197):** `treat` = `TREAT_COST_MON` mon
  → `TREAT_HP_RESTORE` HP (MON-ONLY, no day fallback — without coin
  the verb HIDES) · `rest_sickroom` = 1 day → `REST_SICKROOM_HP`.
  Food is satiety-only (the old cook-heals path is severed).
- **Low-HP work impairment (one-way):** below `LOW_HP_WORK_THRESHOLD`
  of hpMax, work yield+speed take `LOW_HP_WORK_MULT`; labour NEVER
  costs HP.

## 3. Economy

- **Three nouns, never blurred (ADR-107/108/109):** COIN (mon 文,
  spendable) · RICE 🌾 (real resource: eat/store/sell) · KOKU 石高
  (House STANDING — never spent, never carried, immune to loss).
  Confusing them is a canon violation, not a style choice.
- **Rice units (ADR-163):** stored canonically in shō (升) as ONE
  integer in the kura (`banked.rice`); bales (`SHO_PER_BALE`) and koku
  (`SHO_PER_KOKU`, `src/core/constants.ts`) are DISPLAY conversions,
  never separately stored. The player never sees a raw rice integer
  (TST4).
- **Measured-kura two-lane economy (ADR-163):** each labour site has a
  per-season yield POOL (`SITE_POOL_BASE` / `SITE_POOL_DEFAULT`,
  balance.ts) drawn down with diminishing returns (`productionDraw`:
  each act takes a fraction of what REMAINS — grind-proof by
  construction, never fully dries within a season); pools refill at
  season turn (autumn harvest multiplier for season-gated sites). The
  household eats `CONSUMPTION_SHO_PER_DAY` from the kura daily.
- **Seasons are MANUAL containers** (`SEASONS`, constants.ts —
  ADR-150/ADR-153): the six-season wheel Winter → New-Year → Spring →
  Summer → Bon → Autumn is STORED state, advanced only by the
  `advance_season` intent — which is WHY pools refill on season turn
  and the seasonal judge rides every exit. Day-of-week clock only;
  one moon (`LUNAR_PERIOD_DAYS`, a continuous ephemeris).
- **Nengu (年貢)** — Autumn's land tax (`src/core/nengu.ts`, ADR-166):
  `NENGU_KOKU_DEMAND` koku drawn from the kura; the Autumn exit
  REFUSES until the nengu scene reckons it (re-arms yearly); shortfall
  latches `nengu-short` — felt pressure, never numbered in T0.
- **Store-vs-sell:** `RICE_SELL_PRICE_BY_SEASON` swings by season —
  dear in spring, cheap in the autumn glut; the monotonic DIRECTION is
  the lever tests assert. Spoilage (`RICE_SPOILAGE_NUM/DEN`) hits
  carried AND banked separately per season turn — deterministic, no
  RNG, no split-hoard dodge. Kura cap grows with estate stage
  (`KURA_RICE_CAP_BASE/PER_STAGE`).
- **Coin faucet (bounded):** `DAY_WAGE_MON` (`src/core/content/wage.ts`)
  from R5, collected TACTILELY via `collect_wage` — never
  auto-credited; Yohei's market-day purse bounds rice→coin conversion.
- **Vitals — the body split (ADR-178):** `satiety` = per-act work fuel
  ("Body 体", cap grows with level); `hunger` = the slow daily belly
  ("Belly 腹", flat cap). `HUNGER_MEAL_RESTORE == HUNGER_PER_DAY` ON
  PURPOSE — a stocked kura MAINTAINS; only deliberate eating raises.
  Belly's only teeth: rest quality ramps below `HUNGER_FLAT_ABOVE`,
  floored at `HUNGER_REST_FLOOR` — SLOWS, never blocks; no starvation
  death in T0. Satiety throttles work/combat rate to a floor, never 0.
- **Skill economy:** integer 13/10 XP curve (`SKILL_XP_BASE`,
  `SKILL_XP_GROWTH_NUM/DEN`); labour yield scales with skill level,
  capped — accelerates currency WITHOUT trivializing rungs because
  rung progress counts ACTS.
- **Phase-2 pillar engine** (`src/core/pillars.ts`): Estate 家産 grade
  bands `ESTATE_BANDS` (bad/ok/good/great/excellent); T0 ascension
  gate = excellent. Deeds bank only in Phase 2: `ESTATE_DEED_PER_ACT`
  sub-koku fractions × source multipliers (estate-relevant work ONLY —
  forage/woodcut have no source, TST3); per-deed cap ≈ 4% of the good
  band (anti-spike). Seasonal judge = the 3:7 complement of deed
  growth (`SEASONAL_OVER_DEEDS_NUM/DEN`, ±`SEASONAL_SWING`, never
  net-negative). Six-step grade ladder 不可·劣·可·良·優·秀
  (pillars.ts). First ascension boon is always BIG (attribute points +
  overshoot bonus, balance.ts).
- **Determinism invariants:** one seeded RNG family (`src/core/`
  splitmix; no `Math.random` anywhere); economy math is integer
  fixed-point (no floats, no `Math.pow`) so it folds one-day-at-a-time
  with the clock. Breaking either breaks save-replay AND the sim.
