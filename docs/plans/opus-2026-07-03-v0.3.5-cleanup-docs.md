## 1 · ADRs to record (the ratifications)

Seven decisions were **ratified** in the audit — settled intent that belongs in
[`docs/living/decisions.md`](../living/decisions.md) as ADRs so they're not
re-litigated:

- **koku = pure standing, no income flywheel.** koku gates ascension and reads as
  House prestige; it never multiplies income. (Rejected the soft-flywheel.)
- **House Influence lives on the Estate tab** — koku = House standing → the
  house-management tab.
- **Tab kanji glyphs locked** (first cut ratified): Map 地図 · Estate 家 ·
  Inventory 蔵 · Character 己 · Combat 武. *(Note: the Quests-tab glyph is new —
  see §2; and the tab set moves 6→7.)*
- **No morale/upkeep system in housing** — housing comfort stays concrete: rest
  recovery, satiety, and (now) storage. A mood/upkeep stat is explicitly **out of
  scope**.
- **Append-only render engine — all five §6 choices ratified:** migrate all
  surfaces · ship per-phase · the shared `reconcile.ts` helper · full
  flash-elimination · a standing zero-churn test per surface. (With the Now-view
  residual now scheduled to close — build-plan §2.)
- **Rung-beat cast & rewards locked:** keep the three invented faces (Tokubei,
  Rokusuke, Tōzō); rare, varied rewards are allowed (incl. a one-time +1 AGI at
  R3); Naoyuki stays mentioned-but-unseen in T0.

Each ADR gets the next `D-NNN` id, a `created_date: 2026-07-03`, and a one-line
"supersedes/relates" pointer where it firms up a prior provisional call.

## 2 · PRD ripples (Fable + human-signed)

The **overrides** change what the game *is*, so they ripple into
[`docs/living/prd.md`](../living/prd.md):

- **6-tab IA → 7 tabs (Quests promoted).** The locked "6-tab set" is now a 7-tab
  set; the PRD's IA section + any "6 tabs" fact must update, and the new Quests
  tab needs a glyph chosen (a taste call). *(This is the biggest ripple — it
  touches a previously-locked number; run `prd:drift` after the build lands.)*
- **Tab reveal cadence** — Inventory now reveals at R3, not R1 (staggered).
- **Housing semantics** — the hearth homes the **cook verb** (not a stat); the
  chest is **storage** (not a stat); housing comfort = rest/satiety/storage, **no
  morale**. The PRD's housing description must match.
- **Economy** — rice now has a **storage cost** (the store-vs-sell decision is
  real); note the mechanism once the build picks it.
- **Status ladder** — record the T0/T-later split: **T0 grants exactly one home
  status token** (R1→R7); the **full surname → two swords → gōshi ladder is
  T1–T5 planning**. This is a deliberate scope line, not a TODO — the PRD should
  frame status accrual as a multi-tier arc, tiny in T0.
- **R7 capstone** — now a real mechanical branch (was flag-only); the T0→T1 seam
  description updates.
