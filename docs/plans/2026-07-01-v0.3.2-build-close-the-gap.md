## A · v0.3.2 (T0) — pull the T0 build up to the PRD

The fork answers made several of these **T0 now**, not T1.

### A1 · Full combat model: 5 attributes + accuracy/evasion  ·  **HIGH**  ·  D-100
- **What:** grow the 3-attr placeholder (might/guard/vigor — a v0.2 sink-fix) into
  the full **5-attribute** system (STR/AGI/INT/SPD/LUCK, each dual labour+combat,
  D-016); replace flat hit/crit/block (0.9/0.1/0.1) with **accuracy/evasion-driven**
  resolution (PRD §4.6.1/§4.6.3/§4.6.4b) + `DAMAGE_FLOOR = 0.10·atk`.
- **Invariants (must stay RED-able):** first-fight **20–35%** band, graded choice
  curve, **no dominated stance** (D-011/D-016/D-050 — locked).
- **PRD:** §4.4:729–765, §4.6. **Note:** this is the biggest task; sequence it first
  since A2/A3/A6 build on the richer model.

### A2 · Stances = glass cannon ↔ tank  ·  D-101
- **What:** keep `atkMult`/`takenMult` as **the** stance axis (aggressive = more
  damage dealt + more taken; defensive = the inverse; balanced between). **Demote/
  remove `wearMult`** as the stance differentiator — weapon wear stays a durability
  mechanic, not the stance's identity.
- **Invariant:** no-dominant-stance / balanced-not-a-trap (D-050).

### A3 · Third T0 weapon (roster = pole + 2)  ·  D-102
- **What:** add **one more T0 weapon** (build ships 2: carrying-pole + crafted axe).
  Rule is now **"≥1 craftable"** — the 3rd may be found or crafted. Keep the 3-line
  archetype/signature data-model vision (D-026).
- **PRD:** §4.6.9, §1:251, §5:121.

### A4 · Rename the estate purchase ladder E# → U#  ·  D-098
- **What:** rename `estateStage` labels **E1–E4 → U1–U4 ("kura-works")** in code
  (`content/estate.ts`, `render.ts` "E4 · Risen", tests, `t0-content.md` gen). The
  narrative CONDITION ladder stays `E0–E5`. Mechanical rename, no behaviour change.

### A5 · Player vs estate finances  ·  D-099  ·  (design-first)
- **What:** model **two koku lanes** — personal (character purchases at the
  provisioning shop) vs estate (the estate economy). Today's `resources`(carried) vs
  `banked`(kura) is at-risk-vs-safe, **not** player-vs-estate — reconcile the two
  axes. **Start with a short design note** (which pools exist, what the shop spends,
  what the estate ladder spends) before code; part may slip to T1 with the trade
  engine. The T0 provisioning shop itself already ships (bless per D-099).

### A6 · Quest starter set — add HUNT / CLEAR / DEFEND  ·  D-032
- **What:** the `QuestKind` union is `PEST|HUNT|CLEAR` (DEFEND absent) and only
  `pest_crop_raiders` exists. Add HUNT/CLEAR/DEFEND kinds + one T0 quest each — the
  order-free advance-event engine already exists. **PRD:** §1:862.

### A7 · Staggered combat reveals + Bestiary  ·  D-026
- **What:** the R3 reveal currently dumps weapon+auto-resolve+retreat at once. Stage
  it R3 (weapon+auto-resolve+retreat+Bestiary) → R4 (durability+Equipment) → R5
  (stance), one-reveal-per-beat; add a **Bestiary panel**. **PRD:** §3.5.1:509.

### A8 · Estate interior-room reveals  ·  canon §I / §1.12
- **What:** build the Omoya/Main House (R4), Workshops & Granary (R6), lord's study
  (R7) **area reveals** (flavour/reveal-bus beats) — reconcile with the 7-node map
  ceiling (these are area reveals, not new spine nodes). **PRD:** §3.3:290.

### A9 · Richer T0 mob roster  ·  (needs A1)
- **What:** add rice-rats/mamushi + the crop-raiding **monkey troop** (teaches
  multi-target/evasion — depends on A1's richer model). Mamushi-gall consumable is
  T1 (D-095). **PRD:** §5:226.

### A10 · Build-FIX — gate the bandit out of T0  ·  (build has MORE, wrongly placed)
- **What:** the `provisional(v0.2)` bandit (野伏) is reachable at woodlot-edge in T0,
  but the **first HUMAN threat is canon-held for T2** (PRD §5:391, ×10 consistent).
  Remove/relocate/gate it until T2. This is the one item where the *build* overreached
  the PRD — fix the build, not the PRD.
