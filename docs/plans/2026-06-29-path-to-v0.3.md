# Part 2 · Build v0.3 of the game (TypeScript)

> **Scope (roadmap T0-M3 — the #1 gap).** Close the first four-pillar **macro loop** on thin content: stand up
> the live **Estate (家産) pillar** (accrual + seasonal high-water judge) and the **BIG ceremonial manual-opt-in
> T0→T1 ascension** (grade-gate, grade-scaled boon, tier 0→1 stored, schema bump + dev save-wipe) — proving the
> spine actually **CLOSES** — built on the existing R0–R3 demo extended *thinly* to an R7 capstone, **before** any
> T0-M4 showcase breadth. Its coupled precursor is the locked **T0-M2 HP-carry combat retune**.
>
> **Gated on** Part 1's code-affecting decisions. ADRs let it start now (build-first / D-022) at the cost of
> liquid balance numbers possibly re-tuning after playtest — acceptable if balance stays single-sourced in `balance.ts`.

### Carry forward — keep, don't rebuild

- **Pure-core spine:** `reduce(state, intent)` + `tick`/`advanceClock`, immutable structural-sharing `GameState`,
  the universal `applyRewards` bus (whose comment already reserves a Phase-2 `pillarDeltas` hook), the write-once
  unlock/revealPass latch (the *UI-unlocks-incrementally* signature). The spine plugs in — no rewrite.
- **RNG:** seeded splitmix64 with named substreams (combat/loot/**seasonal**/worldgen) + `deriveDayKeyed()`.
- **Clock:** derived `clock.{tick,day}` → season/week/year/lunarPhase (selectors) — the season-boundary judge hooks straight on.
- **Labour ladder:** `accrueRungMeter` + `promoteRungs` AND-gate, threshold map, R0–R3 authored, discover-by-doing
  skills, stamina gates. `phaseOf()` already keys on `rung==='R7' && flags['t0-capstone']` — the contract exists; R4–R7 just aren't authored.
- **Combat:** `resolveFight` race-to-kill sim, analytic + fixed-seed sampled win-rate (seed-robust 20–35% band),
  character level + attribute spend, kendo stance mods, gear/durability/repair, self-recovering soft-setback loss.
- **Economy sinks:** `improve_estate` (koku→stage buffer), `cook_meal`, `spend_attribute` — the linear-koku taste (D-066) is effectively present.
- **The LOCKED House-Influence teaser panel** (`renderHouseInfluence`) — v0.3 **mutates it in place** into the live-Estate + silhouettes view (D-055), not a new build.
- **Save layer:** multi-backend + crash-recovery + an ordered-forward `migrate()` chain (registered-steps map currently empty→identity) + `migrate.test.ts` exercising the contract. The v0.3 bump registers the first real step.
- **DEV harness:** `window.__qa` already exposes `toRung()` (a teleport precursor), `newGame`, `faceWolf`, `fight`, `auto`.
- **Process gates:** 9-gate parallel `verify` (~1.7 s), `pacing:check`, `playcheck` ratchet, content-verifier, the mandatory `diverge` UI gate, milestone-integrity manifest — all green; v0.3 slices attach DoD instruments here.

### Build steps — ordered, spine-first

1. **Schema scaffold.** Add `tier: number` (0..5, stored) + `influence: { estate: { value, highWater } }` to
   `GameState`; bump `SCHEMA_VERSION` 1→2; register the **real** v1→v2 `migrate` step (dev/v0.2 saves WIPED per
   D-067, but author it as the genuine forward path + cover in `migrate.test.ts`).
   — `state.ts`, `constants.ts`, `persistence/migrate.ts(.test.ts)`
2. **R7 capstone (thin).** Widen `RankId` to R7; author R4–R7 as **minimal** rungs (threshold + reveal/reward
   stub, reusing existing labour/combat verbs); fix R3's dead-end `storyGate`; R7's reward sets the `t0-capstone`
   flag that `phaseOf()` already reads to open Phase 2. (Rich R4–R7 breadth is deferred to T0-M4.)
   — `content/ranks.ts`, `content/balance.ts`, `ranks.ts`
3. **Estate pillar accrual.** New `pillars.ts`: grade bands (GOOD/GREAT/EXCELLENT) + an accrue helper; add
   optional `pillarDeltas?` to `RewardBundle`, applied **only** when `phaseOf(state)===2` (deeds bank post-R7
   only); enforce per-deed cap (0.04·band) + the 70% deeds share. — `pillars.ts` (new), `rewards.ts`, `intents.ts`, `state.ts`
4. **Seasonal judged result.** In `advanceClock`, on a **new high-water** at a season boundary, fire a judged
   ±10% Estate result via the day-keyed `seasonal` substream (the 30% seasonal share) + a diegetic log beat. Pure & deterministic.
   — `step.ts`, `pillars.ts`, `rng.ts`
5. **Ascension.** New `ascension.ts`: analytic gate (T0 = Estate ≥ EXCELLENT, D-049/D-057) + `ascensionAvailable`
   selector (unlocks the option, never auto-advances); `ascend` intent sets tier 0→1, grants a grade-scaled
   **permanent** boon (overshoot→better, D-049), reads the D-055 dream/mystery flags to fire the per-tier dream
   beat (fixing the v0.1 write-only-flags bug), emits the ceremonial cascade. First ascension is always big (D-062).
   — `ascension.ts` (new), `intents.ts`, `selectors.ts`, `state.ts`
6. **UI — live Estate + silhouettes + ceremony (⚠ DIVERGE).** Mutate `renderHouseInfluence`: active Estate bar +
   remaining pillars as locked **unnamed** silhouettes (D-055); season-judge readout; `Ascend` CTA gated on
   `ascensionAvailable`; the T0→T1 ceremony (Yūji Syuku title card, silhouettes stir, dream beat, boon reveal).
   These are **new/major surfaces** → the mandatory **`diverge`** contact-sheet (2–3 variants → self-pick →
   R-item) runs before the winner collapses flag-free into main (D-073). — `ui/render.ts`, `ui/styles.css`
7. **T0-M2 combat retune (coupled precursor).** Per D-050: HP **carries** between fights (read
   `state.character.hp`, not `hpMax`); stop resetting HP on win/loss (recover only by eating — couples combat to
   the cook sink); auto-loop eats to recover. **Replace** the dominance-enshrining stance test (`m2.test.ts:119`)
   with a **no-stance-dominated** curve invariant; keep the signed 20–35% single-fight band (D-058). Independent
   of the spine — can land in parallel; slots just before step 3.
   — `combat.ts`, `fight.ts`, `content/balance.ts`, `m2.test.ts`, `app/main.ts`
8. **Retire the DEMO/REAL fork; ship real pacing + DEV tools.** Per D-056: remove `BalanceProfile 'demo'|'real'`
   (+ the `?balance=` / localStorage plumbing) so real **D-049** pacing is the only profile (T0 ~10–15 min/rung,
   floor-exempt). Add a DEV-only 2×/4×/8× time multiplier + extend `__qa.toRung` into a jump-to-rung/tier
   teleport (D-067), both stripped from prod. Re-derive `pacing:check` + `playcheck` baselines against the single profile.
   — `content/balance.ts`, `state.ts`, `app/main.ts`, `ui/render.ts`, `content/ranks.ts`
9. **Tests + gates (milestone-integrity, D-054).** Every DoD instrument resolves to a real test: per-tier
   gate-grade (Estate EXCELLENT opens T0→T1), Phase-2 accrual (post-R7 only, capped, 70/30), seasonal high-water
   judge (deterministic ±10%), the v1→v2 migrate test, ascension tier-bump + boon. Extend `verify-content` to the
   R4–R7 + pillar registries; refresh the `playcheck`/`pacing` baselines; `npm run verify` green per commit.
   — `ranks.test.ts`, `m2.test.ts`, `pillars.test.ts` (new), `ascension.test.ts` (new), `migrate.test.ts`, `verify-content.ts`, `playcheck.baseline.json`

### Explicitly OUT of the v0.3 cut — deferred to T0-M4 breadth

The small walkable map (D-065 — the heaviest T0 build), the first quest, the tiny market, the found/crafted 2nd
weapon (still a Lv2 grant in `fight.ts`), stance/ability reveals. Optional polish that *may* ride alongside but

teleport (D-067) **are** worth pulling in — they directly de-risk playtesting the ascension without hand-grinding.

### Risks

- **Doc-ripple-before-build rework** — the PRD body is still 5-tier & stale; balance magnitudes are liquid
  (D-059) and may re-tune after the ripple/playtest. Keep them in `balance.ts` so a re-tune is cheap, not a rewrite.
- **Schema bump → save-wipe** — adding `influence`+`tier` (v1→v2) wipes dev/v0.2 saves (intended, D-067). The
  inverse risk is treating it as throwaway: the v1→v2 step must be the **real**, test-covered forward path.
- **Spine needs R4–R7 to climb, but those are T0-M4 breadth** — build them **thin** (thresholds + reveal stubs);
  resist pulling quests/market/map/weapon forward or the spine-first cadence collapses into M4.
- **Silent inert pillar** — `phaseOf()` gates accrual on `R7 && t0-capstone`; if R7's flag isn't wired exactly,
  Phase 2 never opens and the loop is silently dead (the v0.1 write-only-flags bug class). Needs an explicit Phase-2-opens test.
- **Fork retirement re-tunes pacing** — `playcheck`/`pacing` baselines + the ≥30-min floor regression must be
  re-derived under the single real profile, or `verify` goes red on the very commit that retires the fork.
- **Two new/major UI surfaces** trigger the mandatory `diverge` gate (D-073) — budget for the contact-sheet +
  R-item; keep main flag-free (branch-preserved losers).
- **Stance-no-dominated needs HP-carry threaded through both forecast and sim** — or the `displayed==tested`
  invariant (D-057) breaks and gedan stays a trap pick (the retune half-lands and re-enshrines dominance).
