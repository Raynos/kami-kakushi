## Process canon this build runs under (2026-06-30 adoption)

Not new scope — the *standing rules* every step below obeys (from D-086…D-088 + the
v0.3-learnings norms). They mostly **sharpen** decisions already in this plan:

- **D-086 · tension > generosity** — the *why* under D-076 (no auto-heal,
  fight-to-death) and D-077 (tight koku): generosity is the thing to **justify**.
  Any "make it comfier" reflex during the build is a decision to surface, not a default.
- **D-088 · e2e + invariants per tier = a hard DoD contract** — T0 already has
  `t0-arc.test` + `invariants.test`; this build must **keep them green and extend
  them** to the new behaviour (they're named in the DoD, milestone-integrity backstops it).
- **Test discipline (AGENTS.md Conventions)** — every new test must be able to go
  **RED**, derive fixtures from the **source of truth** (not magic numbers — the
  M2·8 break), and assert the **design lever** (atk/taken/wear), not a collapsed
  win-rate. Grounds the RED-able tests the steps below call for.
- **a11y soft-check (qa-playtesting §4)** — the DEV panel + every variant are new
  UI surfaces → run a Lighthouse pass **and** a code-level a11y review (disjoint
  classes), on-palette **AND** ≥4.5:1. Soft, not a gate.
- **Diff re-audit before "done" (P1, `battery` diff mode)** — after each
  locked-ADR step (combat, koku, map node), re-audit the **delta** on the current
  build before calling it done; flag + offer-revert if it touched an approved pick (P2).

## Build steps (sequenced — spine of dependencies)

**Step 1 — DEV panel + the variant-toggle infra (the enabler).** A DEV-only
(`import.meta.env.DEV`, stripped from prod) in-UI panel exposing the existing
`__qa` tools as buttons (speed 1/2/4/8×, jump-to-rung/phase/ascension, faceWolf,
auto-targets) **plus a live variant switch** per diverged surface. This unblocks
the human reviewing *any* variant by toggling it in the running build (D-075).
Also: **fix the buggy influence-B variant** so it's comparable. _Open Q: the
variant-coexistence mechanism (a DEV-state `variant: Record<surface,id>` the
renderer reads; nothing in `src/core/**` branches on it)._

**Step 2 — Full variants per surface, each a review item.** Author **2–3 distinct,
working** variants for the surfaces that shipped single-idea ("diverge-LITE",
retired): **craft panel, market panel, Quests tab**; add the existing
**influence (A/B/+C)** and **map (A/B/+C)** into the toggle. **Each variant → its
own line item in `review.md`** (the human picks via the live toggle). Self-pick a
coherent prod default per surface (never blocks). Rewrite the `diverge` skill's
detailed §§ (the branch/screenshot mechanics → the DEV-panel model) here, now that
the panel exists. _Now-canon when authoring variants:_ lean on the **ui-design §9
taste heuristics** (continuous ink > segmented pips; focused diegetic > god's-eye
grid — A19); run the **a11y soft-check** on each variant (D-086…D-088 adoption);
honour **diverge-integrity** (the A19 notes now in the skill — name any corner-cut,
wire the DECISION sheet into the cleanup).

**Step 3 — Combat rework (D-076, grounded by D-086 tension).** Make the fight a
legible **HP-attrition exchange** (you attack ↔ enemy attacks, trade HP to 0 =
death). **Remove the auto-loop's auto-heal** (`main.ts` HP-management) — this *is*
the tension-over-generosity call (D-086): no comfort default. **0 HP = a lost
fight**, and a loss **stops auto-combat**. Eating stays the only mend (D-050) — now
a real pre-fight choice. Re-bless `playcheck`/`pacing` if the win-curve shifts.
_Test-discipline (per the DoD):_ the loss-stops-autopilot test must be **RED-able**;
assert the **design levers** (atk/taken/HP-carry), not a collapsed win-rate (A14);
keep the forecast routed through the **same `mcCombatStats`** the fight uses so the
shown win-rate drops when hurt for free (A6). _Open Q: round-by-round vs summarised
presentation in the log._

**Step 4 — Koku economy (D-077, grounded by D-086 tension).** Keep standing/pillars
**purely deed-based** (no koku→deed coupling). **Add koku sinks** (e.g. a 2nd craft
recipe, a material-funded repair, deeper estate upgrades / market depth — also
clears battery #15 surplus materials) and **retune income vs. sinks** so net koku
stays low through T0–T4 (rich only at T5) — scarcity is the default (D-086).
_Economy invariants to honour (A4, the `economy-arithmetic` lens checks them):_ each
new sink/upgrade must **close a work→output→more-output loop**, not just grant a
*buffer*; watch for **high-water self-inflation** (a payout that re-judges itself).
_Open Q: the exact sink set + the income/sink ratio (liquid)._

**Step 5 — Load-bearing map node (D-078).** A **map node gates a deed source or a
better yield** (walking there unlocks it), tying the map to standing via deeds.
Clears battery #13 (map gates nothing). Fold in the **post-ascension after-state**
(R4#5-adjacent) so the big moment doesn't land on the stale "480/480" panel — this
is now the **F2 canon** ("design the AFTER of every payoff"): the ceremony must
resolve into a satisfying next-state. The battery `fun`/`onboarding` lens runs the
"…and then what?" check on it.

**Step 6 — Clock doc-fix (D-079).** Edit D-053's text to read **active-only-pause**
(drop the "wall-time catch-up" phrasing); align the `main.ts:205` comment. No code
change. Anytime.

**Step 7 — Battery engineering leftovers (no decisions — deferred work).**

  thin T0 spine, or raise the band past the first boundary). _Open Q: which._ _While

- **DOM tests** for the spine + breadth surfaces (battery #11) — jsdom mounts that
  assert each surface's buttons dispatch the right Intent off real `GameState`.
- **DEV-harness smoke test** (battery #12).
- **Dead write-only flags + unrouted stub dialogue cleanup** (battery #19).
- **Milestone-integrity as a verify gate** (battery #20, D-054 — push norm→gate).
  Now **also backstops D-088** (every DoD-named e2e/invariants test must resolve to

- **UI nits:** cold-open "kura-dark" beat (battery #22), the influence grade-word

## Definition of done / gates

- `npm run verify` GREEN throughout; each step its own verify-green commit(s).
- **Every variant in the codebase has a `review.md` line item** (D-075 invariant).
- The DEV panel + all variants are **DEV-only** — verified absent from the prod
  bundle (the `gh-pages.sh` strip-guard, extended to variant ids).
- New behaviour is **RED-able tested** (combat-loss-stops-autopilot, the koku
  sink balance assertion, the seasonal-judge-fires test, the load-bearing node) —
  and each test obeys **test-discipline** (D-086…D-088): fixtures from the source
  of truth (not magic numbers), asserts the **design lever** not a collapsed metric.
- **D-088 — the tier's full-arc e2e + invariants test stay green and EXTEND to the
  new behaviour** (combat attrition, koku sinks, the map-gate, the after-state);
  they are **named in the DoD** and milestone-integrity backstops them.
- **a11y soft-check** on the DEV panel + each variant (Lighthouse + code review,
  on-palette AND ≥4.5:1) — informs, doesn't block.
- **Diff re-audit (P1)** run after each locked-ADR step before it's "done"; a
  touched-an-approved-pick finding is flagged + offered for revert (P2).
- Milestone-integrity (D-054): every DoD line resolves to a real test/tool.

## Then → R1 playtest

The human playtests **after** Step 3 (combat) lands, so they feel the real fight
model, and reviews the variants live via the DEV panel (Step 1–2). The pacing,
koku numbers, and variant picks are the liquid feedback this build is tuned by.
**Before surfacing to the human, run the diff re-audit (P1)** on the current build
and reconcile the hand-off artifacts (screenshots, `review.md`, the playbook —
A23), so R1 lands on an accurate, current picture, not a stale one.
