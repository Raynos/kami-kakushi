## Build steps (sequenced — spine of dependencies)

**Step 1 — DEV panel + the variant-toggle infra (the enabler).** A DEV-only
(`import.meta.env.DEV`, stripped from prod) in-UI panel exposing the existing
`__qa` tools as buttons (speed 1/2/4/8×, jump-to-rung/phase/ascension, faceWolf,
auto-targets) **plus a live variant switch** per diverged surface. This unblocks
the human reviewing *any* variant by toggling it in the running build (D-075).
Also: **fix the buggy influence-B variant** so it's comparable. _Open Q: the
variant-coexistence mechanism (a DEV-state `variant: Record<surface,id>` the
renderer reads; nothing in `src/core/**` branches on it)._

**Step 2 — Full variants per surface, each a review item.** Author **2–3 distinct,
working** variants for the surfaces that shipped single-idea ("diverge-LITE",
retired): **craft panel, market panel, Quests tab**; add the existing
**influence (A/B/+C)** and **map (A/B/+C)** into the toggle. **Each variant → its
own line item in `review.md`** (the human picks via the live toggle). Self-pick a
coherent prod default per surface (never blocks). Rewrite the `diverge` skill's
detailed §§ (the branch/screenshot mechanics → the DEV-panel model) here, now that
the panel exists.

**Step 3 — Combat rework (D-076).** Make the fight a legible **HP-attrition
exchange** (you attack ↔ enemy attacks, trade HP to 0 = death). **Remove the
auto-loop's auto-heal** (`main.ts` HP-management). **0 HP = a lost fight**, and a
loss **stops auto-combat**. Eating stays the only mend (D-050) — now a real
pre-fight choice. Re-bless `playcheck`/`pacing` if the win-curve shifts; add a
RED-able test that a loss halts autopilot. _Open Q: round-by-round vs summarised
presentation in the log._

**Step 4 — Koku economy (D-077).** Keep standing/pillars **purely deed-based** (no
koku→deed coupling). **Add koku sinks** (e.g. a 2nd craft recipe, a material-funded
repair, deeper estate upgrades / market depth — also clears battery #15 surplus
materials) and **retune income vs. sinks** so net koku stays low through T0–T4
(rich only at T5). _Open Q: the exact sink set + the income/sink ratio (liquid)._

**Step 5 — Load-bearing map node (D-078).** A **map node gates a deed source or a
better yield** (walking there unlocks it), tying the map to standing via deeds.
Clears battery #13 (map gates nothing). Fold in the **post-ascension after-state**
(R4#5-adjacent) so the big moment doesn't land on the stale "480/480" panel.

**Step 6 — Clock doc-fix (D-079).** Edit D-053's text to read **active-only-pause**
(drop the "wall-time catch-up" phrasing); align the `main.ts:205` comment. No code
change. Anytime.

**Step 7 — Battery engineering leftovers (no decisions — deferred work).**
- **Seasonal judge fires 0× on the optimal path (battery #8, real bug):** EXCELLENT
  is reached ~day 5 but the first Phase-2 season boundary is ~day 28, so the judge
  never fires before ascension. Fix the cadence (judge per-week / per-N-acts in the
  thin T0 spine, or raise the band past the first boundary). _Open Q: which._
- **DOM tests** for the spine + breadth surfaces (battery #11) — jsdom mounts that
  assert each surface's buttons dispatch the right Intent off real `GameState`.
- **DEV-harness smoke test** (battery #12).
- **Dead write-only flags + unrouted stub dialogue cleanup** (battery #19).
- **Milestone-integrity as a verify gate** (battery #20, D-054 — push norm→gate).
- **UI nits:** cold-open "kura-dark" beat (battery #22), the influence grade-word
  in ink + drop the non-curated 🔒 (battery #21), the per-deed cap binding (#22).

## Definition of done / gates

- `npm run verify` GREEN throughout; each step its own verify-green commit(s).
- **Every variant in the codebase has a `review.md` line item** (D-075 invariant).
- The DEV panel + all variants are **DEV-only** — verified absent from the prod
  bundle (the `gh-pages.sh` strip-guard, extended to variant ids).
- New behaviour is **RED-able tested** (combat-loss-stops-autopilot, the koku
  sink balance assertion, the seasonal-judge-fires test, the load-bearing node).
- Milestone-integrity (D-054): every DoD line resolves to a real test/tool.

## Then → R1 playtest

The human playtests **after** Step 3 (combat) lands, so they feel the real fight
model, and reviews the variants live via the DEV panel (Step 1–2). The pacing,
koku numbers, and variant picks are the liquid feedback this build is tuned by.
