# Philosophy: if it isn't fun, it isn't finished

> Operating-philosophy register **R5**. See the [register](README.md); when a
> tactic conflicts with this, this wins.

**A compiling build is the floor, not the bar.** For this game the real bar is
that it be _paced, genuinely fun, and intentional_ — woodblock/ink, never generic
AI-slop. Function is necessary and nowhere near sufficient.

## The principles

- **Fun is a hypothesis tested by play.** Its _absence_ is measurable —
  instrumented proxies catch boredom, walls, and starvation (necessary, not
  sufficient) — but its _presence_ is never proven by a green proxy. The human is
  the final fun & taste arbiter.
- **Intentional craft over generated defaults.** Lock the design language — the
  mid-Edo woodblock/ink bible, its traditional-instrument audio analogue —
  _before_ building a surface: opinionated constraint reads as handmade, defaults
  read as slop. Restraint (subtraction, significance-gated juice, no spam) is
  itself the craft.
- **The player gets the real game.** Builder conveniences — speed/teleport
  accelerators, the `__qa` dev API, design-variant toggles — live only behind the
  DEV boundary and are stripped from prod (deploy-gate enforced). The player's
  experience is never an artifact of how the agent built or reviewed it.

## Why this exists

- **D-019**: treat fun as the make-or-break priority — a hypothesis tested by
  play, whose absence is measurable.
- `docs/living/fun-factor.md` §3: _"green proxies mean boredom/walls/starvation
  are absent — necessary, not sufficient."_
- `docs/living/ui-design.md` §9 (the anti-slop meta-rule): _"strong, explicit,
