# Session 01 — 2026-06-25 — bootstrap the generic HTML5 game-dev boilerplate

**Summary:** Stood up a lean, game-agnostic Claude Code boilerplate (process/markdown/skills only —
no TypeScript or build tooling yet), modeled on the conventions in `../ironsight-saga`. The game
itself is intentionally not designed yet.

## What changed
- `CLAUDE.md` — how we work (autonomy, bias-to-action, small commits straight to main, resumability,
  Workflows) + conventions (pure-core boundary, one RNG, generate-don't-duplicate, playtest-by-code,
  docs taxonomy) + repo layout.
- `memory/` — `README` (index), `working-agreements.md` (cadence/autonomy/gate), `project-status.md`
  (live snapshot + how to resume).
- `docs/` — `README` stub; `history/README.md` + `history/decisions.md` (ADR log: status legend,
  reversal convention, entry template).
- `human-in-the-loop/` — `README` + `decisions.md` (H-items) + `review.md` (R-items).
- `brainstorms/` — `README` + `PARKED-THREADS.md` (cross-session tangent index).
- `journal/` — `README` + `_TEMPLATE.md` (this file uses it).
- `archive/README.md`, `audit/README.md` — archive + QA-output conventions (audit/ tracked in git).
- `.claude/skills/` — `grill-me` (design discovery) + `capture-game-states` (headless QA capture),
  both generalized from ironsight.
- `.githooks/pre-commit` (journal-entry gate; opt-in) + `.gitignore`.

## Next intended steps

- `docs/`, `archive/`, `audit/` are stub-only (README + templates); fill them as real work lands.
- All design docs are templates with `{{placeholders}}` — no actual game design decided yet.
