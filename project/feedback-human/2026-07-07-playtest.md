### FB-177 · fresh-entries divider missing during live writing — ✅
**Verbatim:** _"I saw new text being written here without the 'new' UI / We
used to have a UI element that rendered for a period of time to seperate
historical logs from new/realtime logs."_
**Reading:** the FB-27 `新 · new` divider regressed for slow batches.
**Root cause** (instrumented live repro): the divider armed its 4.5s fade at
batch ARRIVAL, but the typewriter cascade types a batch out over many
seconds (one long story line alone typed for ~9s) — lines kept landing
after the divider had already faded.
**Fix:** the fade re-arms on every appended/typed line, and the countdown
defers while the cascade is still busy (queue non-empty / typing live) — the
divider now fades 4.5s after the LAST written line. Verified across the full
19s cascade window that previously failed.
**Fixed in:** see this drain's commit.

### F122 · cursor flickers arrow↔hand across the tab-bar gaps — ✅
_(async inbox capture — drain, session `2026-07-07T13-06-19-66f8f3`)_
**Verbatim:** _"Mouse pointer over various clickable elements and buttons
including in the story/progress/chat tabs for the log flicker between pointer
and mouse as you move the mouse. I think the CSS is too fragile here."_
**Reading:** cursor is set per-button, never on the interactive containers, so
every flex-gap between buttons exposes the default arrow — motion across a bar
flips arrow↔hand.
**Root cause** (confirmed live, headless computed-style probe): `.nav`,
`.log-filter-bar`, `.log-filter-group` all compute `cursor: auto` (arrow) with a
3.6px gap between `cursor: pointer` child buttons.
**Fix:** the three interactive tab-row containers claim `cursor: pointer`, so the
gaps + padding no longer show the arrow — no boundary to flicker across (verified
live: containers now compute `pointer`). One shared CSS rule.
**Distilled rule:** an interactive tab-row/button-bar container should claim the
same cursor as its buttons — per-button cursor over a gapped flex row flickers.
**Fixed in:** see this drain's commit.
