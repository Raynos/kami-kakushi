## R0 auto loop — idle while hidden (later capture, ~15:30)

### FB-313 · auto-play stops when the game window isn't focused — ✅
<!-- capture was mis-stamped FB-310 at capture time (collided with the dev-lane's
     FB-310 "Variants tab count"); renumbered to the next free FB-313 on drain. -->


**Verbatim:** _"I noticed that the game in general does not continue to 'Auto
play' when you are not active on the tab. I do want the behavior of the idle
part of the gameplay to continue working if the game is OPEN in a new chrome
window but i'm in VS code or terminal etc. I understand that if i have a chrome
with 5 tabs… it would not continue to idle… But i would like it to be able to
make auto progress without being in focus if its open as a new window with a
single tab."_
**Reading:** the auto loop (`autoStep`) gated on `document.hidden`, and hide
also `cancelAll`'d the in-flight action. On macOS the window-occlusion API
reports a fully-covered dedicated window as `visibilityState: 'hidden'` — the
same bit a truly-backgrounded tab reports, so the platform can't honour the
"dedicated-window keeps going / 5-tabs stops" model directly.
**Fork surfaced → human picked A** (live tick while hidden, over catch-on-return
and park). "Active" now means **tab-ALIVE, not tab-visible**: the loop no longer
gates on `document.hidden` and hide no longer cancels the in-flight action — it
just flushes a save. The browser's own throttling tiers approximate the model
for free (occluded dedicated window ticks at ~1 s; a background tab crawls under
intensive throttling). Progress still ceases entirely on tab-CLOSE (no offline
accrual).
**Doc-update plan:** ADR-174 (refines ADR-148); PRD §2 rippled (the "PAUSES on
`document.hidden`" wording → "hidden-but-open keeps ticking; halts on tab-close").
**Verified:** headless drive with `document.hidden` forced true throughout —
armed auto-rake advanced the game clock (tick 0→2) while hidden (before the
change the `document.hidden` guard held it at 0 by construction).
**Fixed in:** this commit (the FB-313 drain).
