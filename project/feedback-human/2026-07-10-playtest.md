### FB-197 · Space should advance the VN — ✅
**Verbatim:** _"In the VN you can click to advance, but you cant hit space to advance the text, let's advance text with space."_
**Reading:** classic VN keyboard idiom — the scene only listened for clicks
(`render.ts` `root_.addEventListener('click', …)`); no keyboard path existed.
**Fixed in:** (this drain's commit) — a per-scene document `keydown` handler:
**Space + Enter** call the same `introAdvance()` the click uses; skipped when a
control has focus (Space must still press the focused button) or a modal is
open; `preventDefault` stops page scroll; removed on scene teardown. Two RED-able
tests (mid-type advance + the focused-button guard).

### FB-198 · MC speech consistently prefixed "You:" / "Nameless:" — ✅
**Verbatim:** _"In the cold open the 'No.' is not prefixed with You:"_ · _"here
too all text by the main character should be consistntly prefix with you: or
nameless: etc."_ (two captures, one root cause)
**Reading:** the FB-88 rule (every voiced line carries its speaker prefix)
already governs — but MC speech lines were authored as narrator `>` quotes
because the FB-5 grammar had **no player-speech form** (a documented deferral in
intro.md/rung-beats.md).
**Distilled rule:** MC speech is authored `You: "…"` — compiles to
`voice: 'player'` with NO static name; the engine resolves the nameplate at
display time through the G4.7 speaker ladder (`playerSpeaker`: You → Nameless →
Gonbei), so the mid-story label flips land without touching authored lines.
**Fixed in:** (this drain's commit) — the `You:` grammar form (emit + validate +
README spec, RED-able compiler tests); ladder resolution centralized in the
`applyRewards` log funnel + the VN transcript; **all 11** reshaped MC quotes
retagged across intro.md (2), scenes.md (3), rung-beats.md (6); registries +
t0-story.md regenerated.
