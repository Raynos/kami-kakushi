# project/telemetry/ — real-play attended-time reports (F8)

**Local sensor data. Git-ignored by contract — only this README (and the
`.gitignore`) are committed.** Every DEV session auto-drops one `<runId>.md`
report per game run here on session-end (tab hide / unload), overwriting as
the run grows; the DEV panel's Telemetry section has a manual **drop report
now** button. The browser's localStorage ring (`kami.telemetry.v1`) is the
buffer; these files are the archive agents read.

## What a report is

The human's REAL play, measured by the F8 attended-time sessionizer: per-rung
**attended minutes** (5-min-play / 20-min-away / 5-min-play records 10
minutes, not 30) next to the sim's greedy-bot minutes and the design band,
plus economy snapshots, game-tick counts, the segment log, and the **taint
ledger** (`speed>1`, teleports, `qa-drive`, `save-import`… — a tainted run's
time was distorted and is excluded from vs-sim comparisons). The third leg of
the pacing triangle: sim bots (theory) · design bands (intent) · **these
files (reality)**.

## The contract (human-locked 2026-07-05)

1. **Never commit report files.** They are one machine's disposable sensor
   readings, regenerated by simply playing. A fresh clone starts empty —
   that is correct, not a loss.
2. **Agents: read this folder before touching balance.** The D-132 balance
   flow (qa-playtesting.md §2) includes this step: if untainted reports
   exist, quote attended-vs-sim for the touched rungs in the commit body.
3. **The diary rule — distill conclusions into the repo.** A pacing
   CONCLUSION drawn from these files (e.g. "R3 costs 9 attended min vs the
   sim's 4") must land in a committed note — a balance-watch entry or a
   journal section — with the numbers quoted. Raw = local-only insurance;
   the committed distillation is the repo's memory (the `brainstorms/raw/`
   two-tier pattern). Data that stays only here rots when this machine does.
4. **No pruning in v1.** Files are a few KB; revisit only if it ever
   matters.

Transport: `src/telemetry/drop.ts` → the `telemetry-drop` vite dev-middleware
(`src/scripts/telemetry-drop.ts`, `apply: 'serve'` — structurally absent from
builds). Model + report format: `docs/plans/fable-process-F8-play-telemetry.md`
(until the qa-playtesting.md section lands as the living home).
